  • What is DOS Network

    DOS Network is a scalable layer-2 protocol that offers decentralized data feed oracle and decentralized verifiable computation oracle to mainstream blockchains. It connects on-chain smart contracts to off-chain internet data, also supplies unlimited verifiable computation power to blockchains, enabling more commercial applications with real world use cases.

  • What is the role and value of DOS Network (What problems can DOS Network solve?)

    Due to the inherent limitation of existing blockchains’ consensus mechanisms and their deterministic virtual machines, currently there are two biggest issues hindering the widespread adoption of smart contracts and the emergence of large-scale commercial

    • The smart contract on blockchain is a closed and deterministic system and the execution of the smart contract program at any node must have the same deterministic result, so smart contract cannot allow I/O communication with the outside world. Otherwise, the certainty of the execution result cannot be guaranteed, and the blockchain consensus system cannot be achieved. Smart contracts can only get data from closed systems on the chain.

    • There are many resource limitations resulting in the inability to perform complex calculations in smart contracts:

      • Time resources: Smart contracts must be executed within a single block time. Ethereum's block time is about 14 seconds now, EOS is about 0.5 seconds, and normally complex calculations can't be completed in such a short period of time. This is a limitation of time resources.
      • Near real-time and verifiable result: Driven by the secure randomness engine built upon Verifiable Random Function (VRF) and advanced cryptographic primitives, the data request or computation is performed and consented off-chain but verifiable on-chain in a system contract. The delay between receiving request and delivering back response is within two blocks - the optimal one can achieve in theory.
      • Pace resources: Some complex calculations often require a large number of intermediate results, while the space resources on the blockchain are limited and cannot store such a large number of intermediate results. This is a limitation of space resources.
      • Costly: At present, the execution of smart contracts in Ethereum requires consumption of gas, and the amount of gas consumed by complex calculations is high, which is very expensive. In addition, there is a block gas limit on Ethereum, which means that the total amount of gas consumed on a block is limited. At present, it is around 8 million. As a result, the gas that needs to be consumed for some particularly complicated calculations may exceed this limit and cannot be completed at all.
      • The verifier's dilemma: When the calculations in the block are complex and the verifier needs to consume a lot of resources for computational verification, then he will be at a disadvantage in the block operation. And if he does not verify the received block, then he may perform the block operation on a wrong fork, thus falling into a dilemma that it is difficult to make a profit. So we think that the most appropriate way for complex calculations is to delegate to the outside of the chain to complete.

      The role and value of DOS Network is to solve these two problems making same amount of effort to enrich smart contracts and decentralized economy with real world events and more computing powers before blockchain hits mass adoption.

  • What are the advantages and features of DOS Network?

    • Byzantine fault-tolerant decentralized protocol: A trustless solution without single point of failure or downtime, with node runners come and go on the fly.
    • Near real-time and verifiable result: Driven by the secure randomness engine built upon Verifiable Random Function (VRF) and advanced cryptographic primitives, the data request or computation is performed and consented off-chain but verifiable on-chain in a system contract. The delay between receiving request and delivering back response is within two blocks - the optimal one can achieve in theory.
    • Scalable at a low cost: Thanks to the group sharding and off-chain consensus, the total data query throughput and computation capacity increases as more nodes joins DOS network.
    • Compatible to heterogeneous blockchains: With core protocols operating in off-chain clients, it’s easy to support multiple public chains by adding off-chain adaptors and port on-chain governance contracts at a low overhead.
    • Crypto economy: With designed token economic models and cryptographic primitives, it’s highly incentive for node runners to join the network early to earn the “mining” rewards and rapidly grow the network capacity. The network is resistant to sybil attack, with malicious nodes to be detected and forfeited of their stake and honest nodes always to be rewarded.
    • And more: A data feed marketplace could even be built to enable lots of premium (paid) data sources to be able to sell their data service to smart contracts and monetize from blockchain traffic as well. After supporting multiple (at least two) chains, then an upper layer application protocol could be designed on top of DOS NETWORK to support cross chain contract call.

  • What is the structure of DOS Network system?

    DOS Network system is partitioned into two layers with several key components:

    On-chain part: A set of DOS system contracts deployed on supported blockchains, mainly including functionalities such as request handling and response/computation result verification, node registration and staking, stats monitoring, payment processing, etc. On-chain system contracts also provide a universal interface to all user contracts across supported chains

    Off-chain part: A client software implementing the core protocol run by third party users aiming for economic rewards, constituting a distributed network. Client software includes several important modules: event monitoring and chain adaptor module, distributed randomness engine module, off-chain group consensus module, and request processing/computation task processing module depending on the type of oracle service the user node provides.

  • Who can join DOS Network? Why do they want to join DOS Network?

    The off-chain part is a stateless P2P network. This is a Layer 2 network that can support all mainstream blockchains. Anyone can join the network as long as he mortgages a certain amount of tokens and then runs the node's client. After joining the network, you can provide services for users and earn service fees. In addition, in order to stimulate the network cold start, we also introduced the mining mechanism. We reserved 30% of the total token as the reward of network cold start mining. The node operator can obtain the mining reward after joining the network through the mortgaged token.

  • What application scenarios can DOS Network be used in?

    The DOS Network can be used by all applications that need to access out-of-chain data. For example:

    • stable currency, encrypted asset mortgage or algorithmic bank stable currency which needs to obtain real-time price information of cryptocurrency versus legal currency.
    • Market prediction requires real-world information as the final ruling of the forecast.
    • The application about aircraft delays or crop insurance require flight information or weather information as triggers for automatic claims.
    • Gaming games often require secure random numbers. However, in the closed environment of the blockchain, secure random numbers cannot be generated. So secure random numbers can be obtained from outside the chain through the DOS Network.
    • Financial derivatives platforms and lending platforms also require out-of-chain price information as a reference and trigger condition.

    We believe that there is a safe, usable, and efficient decentralized Oracle that encourages more and more application scenarios to be implemented on the blockchain.

  • Do you have your own tokens in this system? What role does it play?

    We have our own token. The node operator joins the network by mortgaging a certain amount of tokens to provide services to users. Users need to pay tokens to these node operators when they need services. At the same time, users also need to pay tokens to the paid data provider after we go online in the future.

  • How does the DOS Network guarantee that the returned data is authentic?

    We use off-chain network consensus to ensure that Oracle has not tampered with the data. Our sub-network is an open peer-to-peer network, which realizes the consensus based on VRF. The random number generated by VRF randomly selects the working group in the network, and the nodes in the group respectively obtain data. When there are more than half of the nodes obtaining the same data, an effective result can be generated and returned the results to the blockchain.

Oracle Problem

Blockchain empowered smart contracts are isolated from the internet world and cannot access external data directly. Computation within the smart contract is also excessively expensive and limited by resource capacity.

DOS Network is

A Decentralized Oracle Service supporting multiple heterogeneous blockchains. DOS Network brings real word data, event and computation power to smart contract in a secure, reliable, efficient and scalable way.

FAQ

  • What is DOS Network?

    DOS Network is a scalable layer-2 protocol that offers decentralized data feed oracle and decentralized verifiable computation oracle to mainstream blockchains. It connects on-chain smart contracts to off-chain internet data, also supplies unlimited verifiable computation power to blockchains, enabling more commercial applications with real world use cases.

  • What problems can DOS Network solve?

    Due to the inherent limitation of existing blockchains’ consensus mechanisms and their deterministic virtual machines, currently there are two biggest issues hindering the widespread adoption of smart contracts and the emergence of large-scale commercial Dapps：

    • The smart contract on blockchain is a closed and deterministic system and the execution of the smart contract program at any node must have the same deterministic result, so smart contract cannot allow I/O communication with the outside world. Otherwise, the certainty of the execution result cannot be guaranteed, and the blockchain consensus system cannot be achieved. Smart contracts can only get data from closed systems on the chain.
    • There are many resource limitations resulting in the inability to perform complex calculations in smart contracts.

    The role and value of DOS Network is to solve these two problems making same amount of effort to enrich smart contracts and decentralized economy with real world events and more computing powers before blockchain hits mass adoption.

  • What is the structure of DOS Network system?

    DOS Network system is partitioned into two layers with several key components:

    • On-chain part: A client software implementing the core protocol run by third party users aiming for economic rewards, constituting a distributed network. Client software includes several important modules: event monitoring and chain adaptor module, distributed randomness engine module, off-chain group consensus module, and request processing/computation task processing module depending on the type of oracle service the user node provides.

  • How does the system defend against witch attacks?

    To avoid this problem, we need node operators to join the network through depositing tokens, which improves the standard to entry, greatly increase the cost of malicious nodes attacking the network and defend against witch attacks. At the same time, because the node operators deposit the tokens in the system, they also have more motivation to maintain the stability of the system because the more stable and prosperous the system, the more valuable the tokens they mortgage. That is to say, we bind the benefits of the node operator and the entire system.

  • Who can join DOS Network? Why do they want to join DOS Network?

    The off-chain part is a stateless P2P network. This is a Layer 2 network that can support all mainstream blockchains. Anyone can join the network as long as he deposits a certain amount of tokens and then runs the node's client. After joining the network, you can provide services for users and earn service fees. In addition, in order to stimulate the network cold start, we also introduced the mining mechanism. We reserved 30% of the total token as the reward of network cold start mining. The node operator can obtain the mining reward after joining the network through depositing tokens.

  • Does DOS Network have tokens in this system? What role does it play?

    We have our own tokens. There are four main reasons why we use our own tokens instead of original currency like ETH:

    • To isolate our system from the native blockchain, such as ETH, to prevent the risk of the native blockchain and currency fluctuations from affecting our service system, making our system relatively stable.
    • To increase the difficulty of attackers. If the attacker wants to buy a large number of tokens to attack our system, then the liquidity of the token will decrease and the price will increase, thus increasing the cost of the attacker.
    • To consolidate the stability and prosperity of the entire ecology with the interests of token holders. If the attacker attacks the network by purchasing a large number of tokens, the attacker himself will suffer losses as the token price decreases.
    • A final approach is provided: the network can be recovered from the attack by forking the DOS token contract. But we will not use this method unless forced.

    The node operator joins the network by depositing a certain amount of tokens to provide services to users. Users need to pay tokens to these node operators when they need services. At the same time, users also need to pay tokens to the paid data provider after we go online in the future.

  • Is DOS Network’s node-selection mechanism based on reputation system?

    No, we are not based on a reputation system in which nodes with good reputation are selected as working nodes with a high probability. Since this will lead to targeting attacks and centralization. It also leads to the Matthew Effect, disincentivizing new node runners with same abilities.

    We completely randomly choose a working group in the network. It is more fair for each node runner in this way.

  • How does the DOS Network guarantee that the returned data is authentic?

    We use off-chain network consensus to ensure that Oracle has not tampered with the data. Our sub-network is an open peer-to-peer network, which realizes the consensus based on VRF. The random number generated by VRF randomly selects the working group in the network, and the nodes in the group respectively obtain data. When there are more than half of the nodes obtaining the same data, an effective result can be generated and returned the results to the blockchain.

  • How can DOS Network return the off-chain data in real time?

    When the user's data request is assigned to the corresponding node, the node will go to the user-specified data source to grab the data, make a consensus within the network and return the result to the user in real time. The whole behavior is automatically completed by the software. No human involvement is required.

  • How is the performance of DOS Network?

    DOS network performance is relatively fast, but how long it takes to return depends on the performance of the layer 1 blockchain. For Ethereum, it is generally possible to return results at the next block time, which is also the fastest return time in theory. For blockchains where the layer 1 backbone is fast, it may take more than one block time to return the results. Regardless of the limitations of the layer 1 main chain, a request can be returned in about 2-5 seconds.
